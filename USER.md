# USER

## 用户输入指南

---

## 一、快速指令（无需标签）

直接用自然语言输入：

```
系统巡检               → 输出完整健康报告（聚合所有可用数据）
系统现在有什么问题     → 同上
今天有没有阻塞         → 分析今天 SQL 阻塞
最近慢SQL集中在哪里    → 慢 SQL 集中度分析
索引健康怎么样         → 索引整体健康报告
服务器压力大吗         → SQL Server + Windows 性能报告
IIS有没有问题          → IIS 健康检查
```

---

## 二、标签输入（精确触发）

### 单模块分析

```
#SQL_BLOCK
（粘贴阻塞数据，或直接说"分析今天阻塞"）

#TABLE_SIZE
（粘贴大表数据，或说"分析表大小"）

#INDEX
（粘贴索引数据，或说"分析今天索引"）

#SLOW_SQL
（上传 slow_query_*.csv，或说"分析今天慢SQL"）

#PERF
（上传 PerfMon CSV，或说"分析今天性能数据"）

#IIS
（粘贴 IIS 数据，或说"分析IIS日志"）

#WINDOWS
（说"检查Windows服务状态"或"查看系统事件日志"）

#PLUGIN
（上传 Plugin .cs / .zip 文件，或说"扫描插件代码"）
```

### 多模块分析

```
#SQL_BLOCK
（阻塞数据）

#PERF
（性能数据）

#IIS
（IIS数据）
```

Agent 会自动聚合多模块输出跨层关联分析报告。

---

## 三、时间表达方式

Agent 支持以下时间语义：

| 输入 | 含义 |
|------|------|
| 今天 | 当前日期 |
| 昨天 | 当前日期 -1 |
| 最近3天 | 今天往前3天 |
| 最近7天 | 今天往前7天 |
| 最近30天 | 今天往前30天 |
| 2026-04-24 | 指定日期 |
| 4月20号到4月24号 | 指定日期范围 |

---

## 四、文件上传支持

支持上传以下文件类型：

| 文件类型 | 用途 |
|----------|------|
| slow_query_*.csv | 慢 SQL 分析 |
| blocking_*.csv | 阻塞历史分析 |
| perf_counter_*.csv | 性能基线分析 |
| index_usage.csv / fragmentation.csv | 索引分析 |
| iis_access_*.log | IIS 日志分析 |
| *.cs / *.zip | Plugin 代码扫描 |
| servers.csv | 更新服务器清单 |
| architecture.png / .pdf | 更新拓扑架构图 |

---

## 五、错误输入处理

若输入格式混乱或标签不正确：

- Agent 会尝试自动识别意图
- 若无法识别，提示重新输入并给出示例
- 若数据不完整，明确列出缺少哪些信息

---

## 六、数据不足时的提示

Agent 会主动提示补充以下信息：

- 阻塞链（sp_whoisactive 输出）
- SQL 执行计划（XML 格式）
- Wait Stats（sys.dm_os_wait_stats）
- PerfMon 数据（如未采集）
- Plugin 源码或反编译代码
- 服务器配置信息（若 environment.json 未初始化）

---

## 七、输出报告下载

分析完成后，可请求：

```
导出报告       → 生成 Markdown 格式完整巡检报告
导出SQL脚本    → 仅导出优化 T-SQL 脚本
导出风险清单   → 仅导出 P1/P2/P3 风险列表
```
