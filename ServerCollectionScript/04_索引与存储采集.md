# 04 - 索引与存储采集

> **目的**：周期性采集每个 CRM 业务库的：
> - 索引使用率（`sys.dm_db_index_usage_stats`）
> - 索引碎片（`sys.dm_db_index_physical_stats`）
> - 缺失索引（`sys.dm_db_missing_index_*`）
> - 表数据量与物理大小（`sys.dm_db_partition_stats`）
> - 索引/主键定义清单（**静态**，库结构变化时更新）
>
> **最终产物**：
> - 按日：`D:\DBA\insagent-data\sql_index\YYYY-MM-DD\{index_usage,missing_index,table_size,index_fragment}.csv`
> - 静态：`D:\DBA\insagent-data\sql_index\index_existing.csv`
>
> agent 的 `sql_index_optimizer` 和 `sql_storage_analysis` 共用此 category。

---

## 一、准备工作

1. 监控账号需要对所有目标数据库有 `VIEW DATABASE STATE` 权限；若是 sa 可忽略。
2. 目标库列表维护在 PowerShell 脚本的 `$Databases` 数组里（也可改为 `SELECT name FROM sys.databases` 自动枚举）。

## 二、SQL 脚本

### 1) 索引使用率 —— `C:\DBA\scripts\Collect-IndexUsage.sql`

```sql
-- 按库执行（由 PowerShell 循环各库 USE）
SELECT
    DB_NAME()                       AS DatabaseName,
    SCHEMA_NAME(o.schema_id)        AS SchemaName,
    OBJECT_NAME(i.object_id)        AS TableName,
    i.name                          AS IndexName,
    i.type_desc                     AS IndexType,
    i.is_unique                     AS IsUnique,
    i.is_primary_key                AS IsPrimaryKey,
    ISNULL(u.user_seeks,     0)     AS UserSeeks,
    ISNULL(u.user_scans,     0)     AS UserScans,
    ISNULL(u.user_lookups,   0)     AS UserLookups,
    ISNULL(u.user_updates,   0)     AS UserUpdates,
    u.last_user_seek                AS LastUserSeek,
    u.last_user_scan                AS LastUserScan,
    u.last_user_update              AS LastUserUpdate,
    p.row_count                     AS RowCount_,
    (p.total_page_count * 8)        AS TotalSizeKB,
    (p.used_page_count  * 8)        AS UsedSizeKB,
    CAST(GETDATE() AS DATE)         AS CollectDate
FROM sys.indexes i
INNER JOIN sys.objects o ON o.object_id = i.object_id AND o.type = 'U'
LEFT  JOIN sys.dm_db_index_usage_stats u
        ON u.object_id = i.object_id AND u.index_id = i.index_id AND u.database_id = DB_ID()
LEFT  JOIN sys.dm_db_partition_stats p
        ON p.object_id = i.object_id AND p.index_id = i.index_id
WHERE i.type > 0                       -- 去掉 HEAP
  AND o.is_ms_shipped = 0
ORDER BY p.row_count DESC;
```

### 2) 索引碎片 —— `C:\DBA\scripts\Collect-IndexFragment.sql`

```sql
SELECT
    DB_NAME()                                     AS DatabaseName,
    SCHEMA_NAME(o.schema_id)                      AS SchemaName,
    OBJECT_NAME(ps.object_id)                     AS TableName,
    i.name                                        AS IndexName,
    i.type_desc                                   AS IndexType,
    ps.partition_number                           AS PartitionNumber,
    ps.index_type_desc,
    ps.avg_fragmentation_in_percent               AS AvgFragmentPct,
    ps.page_count                                 AS PageCount,
    ps.record_count                               AS RecordCount,
    CAST(GETDATE() AS DATE)                       AS CollectDate
FROM sys.dm_db_index_physical_stats(DB_ID(), NULL, NULL, NULL, 'SAMPLED') ps
INNER JOIN sys.indexes i
        ON i.object_id = ps.object_id AND i.index_id = ps.index_id
INNER JOIN sys.objects o
        ON o.object_id = ps.object_id AND o.type = 'U'
WHERE ps.page_count > 500        -- 小表忽略
  AND i.name IS NOT NULL
ORDER BY ps.avg_fragmentation_in_percent DESC;
```

### 3) 缺失索引 —— `C:\DBA\scripts\Collect-MissingIndex.sql`

```sql
SELECT
    DB_NAME(mid.database_id)                          AS DatabaseName,
    OBJECT_SCHEMA_NAME(mid.object_id, mid.database_id) AS SchemaName,
    OBJECT_NAME(mid.object_id, mid.database_id)        AS TableName,
    mid.equality_columns                              AS EqualityColumns,
    mid.inequality_columns                            AS InequalityColumns,
    mid.included_columns                              AS IncludedColumns,
    migs.user_seeks                                   AS UserSeeks,
    migs.user_scans                                   AS UserScans,
    migs.avg_total_user_cost                          AS AvgTotalUserCost,
    migs.avg_user_impact                              AS AvgUserImpactPct,
    migs.avg_total_user_cost * migs.avg_user_impact
        * (migs.user_seeks + migs.user_scans)         AS ImpactScore,
    migs.last_user_seek                               AS LastUserSeek,
    CAST(GETDATE() AS DATE)                           AS CollectDate
FROM sys.dm_db_missing_index_details        mid
JOIN sys.dm_db_missing_index_groups         mig  ON mid.index_handle = mig.index_handle
JOIN sys.dm_db_missing_index_group_stats    migs ON mig.index_group_handle = migs.group_handle
WHERE mid.database_id = DB_ID()
ORDER BY ImpactScore DESC;
```

### 4) 表大小 —— `C:\DBA\scripts\Collect-TableSize.sql`

```sql
SELECT
    DB_NAME()                                   AS DatabaseName,
    s.name                                      AS SchemaName,
    t.name                                      AS TableName,
    p.row_count                                 AS RowCount_,
    (p.reserved_page_count * 8)                 AS ReservedKB,
    (p.used_page_count     * 8)                 AS UsedKB,
    (p.used_page_count     * 8) / 1024          AS UsedMB,
    (p.reserved_page_count * 8) / 1024 / 1024   AS ReservedGB,
    t.create_date                               AS CreateDate,
    t.modify_date                               AS ModifyDate,
    CAST(GETDATE() AS DATE)                     AS CollectDate
FROM sys.tables t
INNER JOIN sys.schemas s ON s.schema_id = t.schema_id
INNER JOIN (
    SELECT object_id,
           SUM(row_count)                                                AS row_count,
           SUM(reserved_page_count)                                      AS reserved_page_count,
           SUM(used_page_count)                                          AS used_page_count
    FROM sys.dm_db_partition_stats
    WHERE index_id IN (0, 1)       -- HEAP 或 CLUSTERED
    GROUP BY object_id
) p ON p.object_id = t.object_id
WHERE t.is_ms_shipped = 0
ORDER BY p.reserved_page_count DESC;
```

### 5) 索引清单（静态）—— `C:\DBA\scripts\Collect-IndexExisting.sql`

```sql
-- 仅在库结构变更后执行，覆盖 index_existing.csv
SELECT
    DB_NAME()                                   AS DatabaseName,
    SCHEMA_NAME(o.schema_id)                    AS SchemaName,
    OBJECT_NAME(i.object_id)                    AS TableName,
    i.name                                      AS IndexName,
    i.type_desc                                 AS IndexType,
    i.is_unique,
    i.is_primary_key,
    i.is_disabled,
    STUFF((
        SELECT ', ' + c.name +
               CASE WHEN ic.is_descending_key = 1 THEN ' DESC' ELSE '' END
        FROM sys.index_columns ic
        INNER JOIN sys.columns c
             ON c.object_id = ic.object_id AND c.column_id = ic.column_id
        WHERE ic.object_id = i.object_id
          AND ic.index_id  = i.index_id
          AND ic.is_included_column = 0
        ORDER BY ic.key_ordinal
        FOR XML PATH(''), TYPE).value('.', 'nvarchar(max)'), 1, 2, '')    AS KeyColumns,
    STUFF((
        SELECT ', ' + c.name
        FROM sys.index_columns ic
        INNER JOIN sys.columns c
             ON c.object_id = ic.object_id AND c.column_id = ic.column_id
        WHERE ic.object_id = i.object_id
          AND ic.index_id  = i.index_id
          AND ic.is_included_column = 1
        FOR XML PATH(''), TYPE).value('.', 'nvarchar(max)'), 1, 2, '')    AS IncludedColumns
FROM sys.indexes i
INNER JOIN sys.objects o ON o.object_id = i.object_id AND o.type = 'U'
WHERE i.type > 0
  AND o.is_ms_shipped = 0
ORDER BY TableName, IndexName;
```

## 三、一键导出 PowerShell —— `C:\DBA\scripts\Export-IndexStorage-Csv.ps1`

```powershell
param(
    [string]$Server    = "localhost",
    [string[]]$Databases = @("hcrm_MSCRM","hcrm_MSCRM_CONFIG"),   # 目标库
    [string]$StatDate  = $(Get-Date -Format "yyyy-MM-dd"),
    [string]$RootDir   = "D:\DBA\insagent-data\sql_index",
    [switch]$StaticOnly          # 仅更新 index_existing.csv
)

$DayDir = Join-Path $RootDir $StatDate
if (-not $StaticOnly -and -not (Test-Path $DayDir)) {
    New-Item -ItemType Directory -Path $DayDir | Out-Null
}
if (-not (Test-Path $RootDir)) {
    New-Item -ItemType Directory -Path $RootDir | Out-Null
}

function Export-PerDb($name, $sqlFile, $outCsv) {
    $all = @()
    foreach ($db in $Databases) {
        Write-Host "[$name] $db ..."
        $rows = Invoke-Sqlcmd -ServerInstance $Server -Database $db -InputFile $sqlFile -QueryTimeout 0
        if ($rows) { $all += $rows }
    }
    if ($all.Count -gt 0) {
        $all | Export-Csv -Path $outCsv -NoTypeInformation -Encoding UTF8
        Write-Host "  -> $outCsv ($($all.Count) rows)"
    } else {
        Write-Host "  (empty)"
    }
}

if ($StaticOnly) {
    Export-PerDb "index_existing" "C:\DBA\scripts\Collect-IndexExisting.sql" `
                 (Join-Path $RootDir "index_existing.csv")
    exit 0
}

Export-PerDb "index_usage"    "C:\DBA\scripts\Collect-IndexUsage.sql"    (Join-Path $DayDir "index_usage.csv")
Export-PerDb "index_fragment" "C:\DBA\scripts\Collect-IndexFragment.sql" (Join-Path $DayDir "index_fragment.csv")
Export-PerDb "missing_index"  "C:\DBA\scripts\Collect-MissingIndex.sql"  (Join-Path $DayDir "missing_index.csv")
Export-PerDb "table_size"     "C:\DBA\scripts\Collect-TableSize.sql"     (Join-Path $DayDir "table_size.csv")

Write-Host "DONE -> $DayDir"
```

## 四、调度建议

| 任务 | 频率 | 命令 |
|---|---|---|
| 索引使用率 / 碎片 / 缺失索引 / 表大小 | **每周一次**（周日凌晨） | `Export-IndexStorage-Csv.ps1` |
| 索引清单（静态） | **库结构变更后** 或 **每月一次** | `Export-IndexStorage-Csv.ps1 -StaticOnly` |

> `dm_db_index_usage_stats` 会在 SQL Server 重启后清零，所以周期不宜过短，一周一次平衡数据稳定性与时效性。

## 五、部署调度（任务计划程序）

在目标 SQL 服务器管理员 PowerShell 执行：

```powershell
# ---------- Task 1: 周采集 → 每周日 02:00 ----------
$action  = New-ScheduledTaskAction -Execute 'powershell.exe' `
           -Argument '-NoProfile -ExecutionPolicy Bypass -File C:\DBA\scripts\Export-IndexStorage-Csv.ps1'
$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Sunday -At 2:00AM
$principal = New-ScheduledTaskPrincipal -UserId 'SYSTEM' -RunLevel Highest
$settings  = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries `
             -StartWhenAvailable -ExecutionTimeLimit (New-TimeSpan -Hours 2)
Register-ScheduledTask -TaskName 'InsAgent_Export_IndexStorage' `
    -Action $action -Trigger $trigger -Principal $principal -Settings $settings -Force

# ---------- Task 2 (可选): 静态索引清单 → 每月 1 号 01:30 ----------
$action  = New-ScheduledTaskAction -Execute 'powershell.exe' `
           -Argument '-NoProfile -ExecutionPolicy Bypass -File C:\DBA\scripts\Export-IndexStorage-Csv.ps1 -StaticOnly'
$trigger = New-ScheduledTaskTrigger -Once -At (Get-Date).Date.AddDays(1).AddHours(1).AddMinutes(30) `
           -RepetitionInterval (New-TimeSpan -Days 30) `
           -RepetitionDuration ([TimeSpan]::MaxValue)
Register-ScheduledTask -TaskName 'InsAgent_Export_IndexExisting' `
    -Action $action -Trigger $trigger -Principal $principal -Settings $settings -Force
```

> Windows 任务计划没有原生「每月 N 号」选项，上面用 `-RepetitionInterval 30 days` 作近似。若需严格每月 1 号，可改用「任务计划程序」GUI 或 `schtasks /create /sc monthly /d 1`。

### 数据保留

- 日期子目录（`YYYY-MM-DD\*.csv`）：由 **README 统一的 `Clean-OldData.ps1`** 清理，默认 `sql_index` 保留 **90 天**（约 12 周的历史数据）。
- `index_existing.csv`（category 根目录的静态文件）**不会被清理**，每次采集覆盖。
- 自定义保留 30 天样例：
  ```powershell
  powershell -File C:\DBA\scripts\Clean-OldData.ps1 -RetentionDays @{sql_index=30}
  ```
